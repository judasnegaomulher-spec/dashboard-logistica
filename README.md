# 📦 Dashboard Inteligente de Monitoramento Logístico

## 📖 Sobre o Projeto

Este projeto foi desenvolvido para atender ao desafio proposto por uma empresa nacional de logística que enfrenta dificuldades para monitorar atrasos em entregas realizadas em diferentes regiões do país.

O objetivo do dashboard é transformar dados operacionais em informações estratégicas, permitindo aos gestores identificar rapidamente gargalos logísticos, regiões críticas e transportadoras com maior incidência de atrasos.

---

# 🎯 Problema de Negócio

Atualmente a empresa depende de planilhas extensas e relatórios manuais, dificultando:

- Identificação de transportadoras com maior índice de atraso;
- Visualização de regiões críticas;
- Priorização de entregas problemáticas;
- Acompanhamento de tendências operacionais;
- Tomada de decisão em tempo real.

---

# 💡 Solução Desenvolvida

Foi criado um Dashboard Inteligente de Monitoramento Logístico utilizando:

- HTML5
- CSS3
- JavaScript
- Chart.js
- GitHub Pages

A solução centraliza as informações operacionais em uma única interface interativa e visual.

---

# 📊 Indicadores (KPIs)

O dashboard apresenta:

### Total de Entregas
Quantidade total de entregas analisadas.

### Entregas Atrasadas
Quantidade de entregas cujo prazo foi ultrapassado.

### Percentual de Atraso
Percentual de entregas atrasadas em relação ao total.

### Maior Atraso
Maior quantidade de dias de atraso encontrada na base.

---

# 📈 Recursos de Análise

O dashboard possui:

## Filtros Interativos

- Filtro por Região
- Filtro por Transportadora

Todos os indicadores e gráficos são atualizados automaticamente.

---

## Comparação entre Transportadoras

Gráfico de barras que apresenta a quantidade de atrasos por transportadora.

Objetivo:

Identificar quais transportadoras possuem maior incidência de problemas operacionais.

---

## Análise por Região

Gráfico Doughnut (Rosca) mostrando a distribuição dos atrasos por região.

Objetivo:

Identificar regiões críticas para atuação prioritária.

---

## Ranking de Entregas Críticas

Gráfico horizontal com as entregas ordenadas do maior para o menor atraso.

Objetivo:

Facilitar a priorização das ocorrências mais urgentes.

---

## Tendência Operacional

Gráfico de linha demonstrando a evolução dos atrasos identificados na base.

Objetivo:

Auxiliar na percepção de padrões operacionais.

---

## Alertas Automáticos

O sistema identifica automaticamente:

- Entrega com maior atraso;
- Transportadora mais crítica;
- Região mais crítica.

Essas informações são destacadas visualmente para acelerar a tomada de decisão.

---

## Tabela Dinâmica

A tabela exibe exclusivamente entregas atrasadas e apresenta:

- ID da entrega;
- Transportadora;
- Região;
- Prazo previsto;
- Dias reais;
- Dias de atraso;
- Nível de prioridade.

---

# 🚦 Critérios de Priorização

As entregas são classificadas automaticamente:

### 🔴 Alta Prioridade
Atraso igual ou superior a 5 dias.

### 🟡 Média Prioridade
Atraso entre 3 e 4 dias.

### 🟢 Baixa Prioridade
Atraso inferior a 3 dias.

---

# 🧮 Regra de Negócio Utilizada

Uma entrega é considerada atrasada quando:

dias_reais > prazo_dias

O atraso é calculado pela fórmula:

atraso = dias_reais - prazo_dias
